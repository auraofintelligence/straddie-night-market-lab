Disruption and backup checks

Questions for changed timetables, full services and weather.

A useful transport page needs the normal timetable plus the backup path. These questions stop a listing from becoming stale the moment a boat, bus or weather window changes.

Service alerts

  • Has the SeaLink status page been checked today?
  • Has the bus service update page or Translink journey planner changed?
  • Has the Flyer office or timetable page changed today?

Late-return backup

  • What is the last realistic ferry and bus combination?
  • Which taxi, StradEasy or private shuttle option is actually available?
  • Who can update the listing if the last return changes?

Full-service backup

  • What advice is shown if a ferry, bus, taxi or shuttle is full?
  • Is there a lower-pressure arrival window?
  • Can visitors be pointed to overnight-stay or wait-and-return options?

Weather backup

  • Which weather condition changes the market plan?
  • Does the host have a wet-weather location or cancellation cutoff?
  • Which message goes on the public page first?

Operator contact

  • Who confirms extra services, charters or late transfer availability?
  • What details can be repeated publicly without creating confusion?
  • When was the answer last checked?

Venue fallback

  • Can the venue hold people safely if a return service is delayed?
  • Is there seating, water, toilets and clear staff information?
  • What is the calmest visitor message?

Public display

Clear enough for a visitor, structured enough for a system.

The public wording can stay simple: current timetable, route, next boat, next bus, whether booking is needed, what changed, and the safest return options. The same record also creates learning data for the next market.
